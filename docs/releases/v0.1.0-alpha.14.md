# v0.1.0-alpha.14

Replacement alpha prerelease for early testers.

Canonical tracked changelog entry: `CHANGELOG.md`

## What's Included

- a fix so the dashboard no longer shows numeric-only placeholder heroes such
  as `1` or `2` as if they were readable hero labels
- a fix so import, review, and final save now block numeric-only `hero`
  placeholders instead of letting them persist into new saved matches

## Supported Scope

- one supported Simplified Chinese post-match detailed-data screenshot
- local screenshot import
- on-device processing
- required review before final save

Unsupported screenshots are rejected.

## Installation

Download the attached Android release APK from this GitHub prerelease and
install it on your Android device.

If you already installed `v0.1.0-alpha.13`, install this newer APK over it.

If Android blocks installation from the downloaded source, enable installation
for that source and retry.

## Diagnostics Support

If you hit a problem in this build, open `Diagnostics` in the app and copy the
export into your support message. `Current Version` is visible near the top,
and the diagnostics surface shows readable failure metadata such as `Reason`,
`Surface`, and OCR text when available. The export still excludes the original
screenshot and full saved record payload.

## Known Limitations

- Hero may still require manual entry during review.
- This is an alpha prerelease, not a stable `1.0.0` release.
- The app does not support additional templates or non-Chinese screenshots in
  this release.

## Privacy

The app is local-first and processes screenshots on-device.
